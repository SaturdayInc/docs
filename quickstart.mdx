# Quickstart

This guide gets you from zero to a working nutrition calculation in under 5 minutes.

## 1. Get your API key

Contact [api@saturday.fit](mailto:api@saturday.fit) to request a sandbox API key. Sandbox keys start with `sk_test_` and work against test data with no rate limit concerns.

```
sk_test_abc123def456...
```

<Note>
  Sandbox keys return realistic data but never affect production athletes. Use them freely during development.
</Note>

## 2. Make your first calculation

The core of Saturday's API is the nutrition calculation endpoint. It takes activity parameters and returns a fuel prescription.

<CodeGroup>

```bash cURL
curl -X POST https://api.saturday.fit/v1/nutrition/calculate \
  -H "Authorization: Bearer sk_test_abc123def456" \
  -H "Content-Type: application/json" \
  -d '{
    "activity_type": "running",
    "duration_minutes": 90,
    "intensity": "moderate",
    "athlete_weight_kg": 70,
    "conditions": {
      "temperature_celsius": 25,
      "humidity_percent": 60
    }
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.saturday.fit/v1/nutrition/calculate",
    headers={
        "Authorization": "Bearer sk_test_abc123def456",
        "Content-Type": "application/json",
    },
    json={
        "activity_type": "running",
        "duration_minutes": 90,
        "intensity": "moderate",
        "athlete_weight_kg": 70,
        "conditions": {
            "temperature_celsius": 25,
            "humidity_percent": 60,
        },
    },
)

data = response.json()
print(data)
```

```typescript TypeScript
const response = await fetch(
  "https://api.saturday.fit/v1/nutrition/calculate",
  {
    method: "POST",
    headers: {
      Authorization: "Bearer sk_test_abc123def456",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      activity_type: "running",
      duration_minutes: 90,
      intensity: "moderate",
      athlete_weight_kg: 70,
      conditions: {
        temperature_celsius: 25,
        humidity_percent: 60,
      },
    }),
  }
);

const data = await response.json();
console.log(data);
```

</CodeGroup>

## 3. Read the response

A successful response returns a fuel prescription with safety metadata:

```json
{
  "prescription": {
    "carbohydrate": {
      "grams_per_hour": 60,
      "total_grams": 90,
      "confidence": "high"
    },
    "hydration": {
      "ml_per_hour": 600,
      "total_ml": 900,
      "confidence": "moderate"
    },
    "sodium": {
      "mg_per_hour": 500,
      "total_mg": 750,
      "confidence": "moderate"
    }
  },
  "safety": {
    "risk_level": "low",
    "flags": [],
    "warnings": [],
    "guardrails_applied": ["sodium_ceiling", "fluid_physiological_limit"]
  },
  "metadata": {
    "request_id": "req_abc123def456",
    "calculation_version": "2024.1",
    "inputs_quality": "minimal",
    "enrichment_available": true
  }
}
```

Key fields to understand:

| Field | What it means |
|-------|---------------|
| `prescription.*` | The actual fuel targets — what to eat, drink, and supplement |
| `prescription.*.confidence` | How confident Saturday is in this number (`high`, `moderate`, `low`) |
| `safety.risk_level` | Overall risk assessment for this prescription |
| `safety.guardrails_applied` | Which safety limits were checked or enforced |
| `metadata.inputs_quality` | How complete the inputs were (`minimal`, `standard`, `comprehensive`) |
| `metadata.enrichment_available` | Whether providing more data would improve accuracy |

<Warning>
  **Safety data is never gated.** Every response includes full safety metadata regardless of subscription status. Bad hydration can cause hyponatremia — a potentially fatal condition. Safety information is always free and complete.
</Warning>

## 4. Understand teaser vs. full responses

The response above shows **full precision** numbers because it's from a sandbox key. In production, responses depend on the athlete's subscription status:

| Athlete status | What they get | Example |
|----------------|---------------|---------|
| **Subscribed** | Exact numbers | `"grams_per_hour": 60` |
| **Free / teaser** | Ranges | `"grams_per_hour_range": { "min": 45, "max": 75 }` |

Teaser responses include a `subscription` field with a CTA:

```json
{
  "subscription": {
    "status": "teaser",
    "upgrade_url": "https://saturday.fit/subscribe?ref=YOUR_PARTNER_ID",
    "message": "Subscribe to Saturday for exact fueling targets"
  }
}
```

See [Freemium Model](/guides/freemium-model) for the full details.
